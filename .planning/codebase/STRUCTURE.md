# Codebase Structure

**Analysis Date:** 2026-04-13

## Directory Layout

```text
ai-service-now-vision/
├── src/                  # Application source (App Router, domain services, DB, shared libs)
├── drizzle/              # SQL migrations and Drizzle migration metadata snapshots
├── scripts/              # Local setup/seed/ops scripts
├── tests/                # Unit/integration-style Vitest suites
├── e2e/                  # Playwright end-to-end suites and helpers
├── docs/                 # Product/setup/integration docs
├── supabase/             # Supabase setup notes and local workflow docs
├── public/               # Static assets served by Next.js
├── package.json          # Script/dependency manifest
├── next.config.ts        # Next.js runtime + security header config
└── drizzle.config.ts     # Drizzle CLI database/migration configuration
```

## Directory Purposes

**`src/app`:**
- Purpose: Next.js App Router boundary for pages, layouts, API routes, and server actions.
- Contains: Route segments (`(dashboard)`), auth pages, `api` handlers, `actions` modules.
- Key files: `src/app/layout.tsx`, `src/app/page.tsx`, `src/app/(dashboard)/layout.tsx`, `src/app/api/v1/requests/route.ts`, `src/app/actions/requests.ts`

**`src/server`:**
- Purpose: Domain workflows and side-effect orchestration.
- Contains: Request lifecycle, routing, fulfillment queue, connectors, policy checks, webhooks, audit, notifications.
- Key files: `src/server/create-request.ts`, `src/server/request-decision.ts`, `src/server/fulfillment-queue.ts`, `src/server/webhooks.ts`, `src/server/change-ticket.ts`

**`src/lib`:**
- Purpose: Cross-cutting infrastructure helpers shared by route/actions/services.
- Contains: Auth/session wrappers, env parsing, schema validators, crypto/encryption, SDK wrappers.
- Key files: `src/lib/auth.ts`, `src/lib/session.ts`, `src/lib/env.ts`, `src/lib/request-schemas.ts`, `src/lib/api-sdk.ts`

**`src/db`:**
- Purpose: Database access and schema declarations.
- Contains: Drizzle schema modules and `pg` pool/drizzle instance.
- Key files: `src/db/app-schema.ts`, `src/db/auth-schema.ts`, `src/db/index.ts`, `src/db/schema.ts`

**`src/components`:**
- Purpose: Reusable UI components used by App Router pages.
- Contains: Navigation, request hub widgets, admin helpers, auth UI utilities.
- Key files: `src/components/dashboard-nav.tsx`, `src/components/requests-hub.tsx`, `src/components/toast.tsx`

**`src/data`:**
- Purpose: Seed/template data consumed by onboarding/admin flows.
- Contains: JSON catalog templates.
- Key files: `src/data/catalog-templates/it-startup.json`, `src/data/catalog-templates/healthcare-lite.json`

**`drizzle`:**
- Purpose: Migration SQL and snapshots generated by Drizzle.
- Contains: Versioned migration files and `meta` snapshots.
- Key files: `drizzle/0003_routing_email_approval.sql`, `drizzle/meta/_journal.json`

**`tests` and `e2e`:**
- Purpose: Automated verification layers.
- Contains: Vitest unit tests and Playwright browser tests with helpers.
- Key files: `tests/ai-request-guard.test.ts`, `tests/catalog-categories.test.ts`, `e2e/change-ticket-flow.spec.ts`, `e2e/helpers/pg.ts`

## Key File Locations

**Entry Points:**
- `src/app/layout.tsx`: Root HTML shell and global styles
- `src/app/page.tsx`: Unauthenticated landing page and session redirect gate
- `src/app/(dashboard)/layout.tsx`: Authenticated dashboard chrome + nav
- `src/app/api/auth/[...all]/route.ts`: Better Auth route adapter
- `src/app/api/internal/worker/fulfillment/route.ts`: Scheduled worker endpoint

**Configuration:**
- `package.json`: Scripts, dependencies, test/build/lint commands
- `next.config.ts`: Security headers and Next runtime options
- `tsconfig.json`: Strict TypeScript + `@/*` alias mapping
- `drizzle.config.ts`: Drizzle schema path, output folder, DB credentials
- `playwright.config.ts`: E2E runner and app boot behavior
- `vitest.config.ts`: Unit test runner config

**Core Logic:**
- `src/server/create-request.ts`: Request creation orchestration
- `src/server/request-decision.ts`: Approval decision + fulfillment enqueue
- `src/server/fulfillment-queue.ts`: Durable fulfillment job processing
- `src/server/webhooks.ts`: Durable webhook delivery/retry
- `src/server/change-ticket.ts`: Change workflow state machine logic
- `src/server/policy-engine.ts`: Allow/deny policy boundary

**Testing:**
- `tests/*.test.ts`: Vitest suites for server/lib behavior
- `e2e/*.spec.ts`: Playwright UI + API journey tests
- `e2e/helpers/*.ts`: Shared E2E data/bootstrap utilities

## Naming Conventions

**Files:**
- Use kebab-case filenames for modules and components: `request-decision.ts`, `change-ticket-panel.tsx`.
- Use Next.js route conventions for page boundaries: `page.tsx`, `layout.tsx`, `route.ts`.
- Use bracket syntax for dynamic route params: `src/app/(dashboard)/requests/[id]/page.tsx`.

**Directories:**
- Use route-group folders for layout grouping: `src/app/(dashboard)`.
- Group by concern in backend modules: `src/server/connectors`, `src/server/notifications`, `src/server/ai`.
- Keep API versioning explicit in path: `src/app/api/v1/*`.

## Where to Add New Code

**New Feature:**
- Primary code: `src/server/<feature>.ts` for domain logic + `src/app/(dashboard)/<feature>/page.tsx` (or other route) for UI.
- Tests: `tests/<feature>.test.ts` for unit/domain logic and `e2e/<feature>-flow.spec.ts` for full workflow.

**New Component/Module:**
- Implementation: `src/components/<component-name>.tsx` when shared across routes; colocate route-only client components under the owning route directory in `src/app`.

**Utilities:**
- Shared helpers: `src/lib/<utility>.ts` when reused across app/server boundaries.
- DB-related shared utilities: `src/db` if utility touches schema/persistence concerns.

## Special Directories

**`.planning`:**
- Purpose: Planning artifacts and codebase maps used by GSD workflows.
- Generated: Yes (agent-generated docs and process artifacts)
- Committed: Yes (intended to support future planning/execution)

**`.next`:**
- Purpose: Next.js build output cache.
- Generated: Yes
- Committed: No

**`node_modules`:**
- Purpose: Installed dependencies.
- Generated: Yes
- Committed: No

**`test-results`:**
- Purpose: Playwright output artifacts.
- Generated: Yes
- Committed: No
